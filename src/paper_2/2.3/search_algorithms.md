## Linear Search
The linear search algorithms searches an array in a __linear fashion__.

This means that it checks the first element, then the second, third, etc.

Therefore, linear search has an average (and worst case) time complexity of O(n), since the time grows at the same rate as the dataset grows.


> [!NOTE]
> It is good to know the best case time complexities of algorithms. For linear search this is O(1)
> This is because the best case is the searched for element being the first in the list.

### Pros of Linear Search
- Easy to implement
- Quick for small lists

### Cons of Linear Search
- Inefficient (very slow with big lists)
